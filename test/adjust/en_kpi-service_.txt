																Element:h3
																Element:span
																	1
																	Versioning
																Element:p
																	The API versions major releases. To avoid confusion, the API version is explicitly written in the URL paths for all endpoints. For example, you’ll be accessing
																Element:code
																	http://api.adjust.com/kpis/v1/..
																	for version 1.
																Element:h3
																Element:span
																	2
																	Response Formats
																Element:p
																	The API supports two response formats - CSV and JSON, with JSON as default. To specify a different format you can either suffix the endpoints accordingly (e.g. add
																Element:code
																	.csv
																	at the end of endpoint path) or send an
																Element:code
																	Accept: text/csv
																	HTTP header with your request. Throughout this document we mainly use JSON for exemplifying requests and responses.
																Element:h3
																Element:span
																	3
																	Authentication
																Element:p
																	Your access to the KPI Service is tied to your Adjust user account. Each user account has an associated user token, to allow you to individually control access to your KPIs.
																Element:p
																	You can find your own user token in the dashboard, under “Account Settings > Your Data”. This is the user token we’ll be referring to for your authentication below.
																Element:p
																	Each user token has the same limitations as the dashboard user. That is, if you set
																Element:a
																	custom user permissions
																	, their access via the KPI Service will follow the same rules as their dashboard access.
																Element:p
																	You can at any time renew your user token.
																Element:p
																	To authenticate against the API using your user token you have two options and you can pick whichever one you prefer for a given use case:
																Element:ul
																Element:li
																	As an HTTP GET parameter - append
																Element:code
																	user_token=your_user_token
																	to any API query.
																Element:li
																	As an HTTP Authentication header - add the following HTTP header:
																Element:pre
																	`Authorization: Token token=your_user_token`
																Element:p
																	All examples in this document assume an HTTP Header authentication, but you can easily append a GET parameter with the correct token to the examples and they would still be valid.
																Element:h3
