				Element:hr
					In case you were wondering which one of these two to use, this tutorial will cover the difference between logs and delivery reports. Logs and reports carry similar information about sent messages. Once you retrieve a delivery report, you will not be able to get the same report again. As opposed to delivery reports, logs can be requested more than once. [block:callout] { "type": "warning", "body": "**Logs** should be used for statistical analysis only.\n\n**Delivery reports** should be used for **message delivery status information purposes**. They can be collected over the **API call** or received in **real time** over **HTTP forward method** by supplying your `notifyUrl` when sending a message. \n\nHTTP forward method is the most efficient way to access delivery reports since each delivery report is forwarded to your address **as soon as status becomes available** and it is the recommended method for collecting message status information.", "title": "Important" } [/block] [block:api-header] { "type": "basic", "title": "Delivery reports: HTTP forward example" } [/block] Receive Delivery report for your sent messages in real time by specifying your unique URL where Delivery reports will be forwarded via HTTP POST request. Use fully featured SMS method to define `notifyUrl` like in this example: [block:code] { "codes": [ { "code": "POST /sms/1/text/advanced HTTP/1.1\nHost: api.infobip.com\nAuthorization: Basic QWxhZGRpbjpvcGVuIHNlc2FtZQ==\nContent-Type: application/json\n\n{\n \"messages\": [\n {\n \"destinations\": [\n {\n \"to\": \"38598111\"\n }\n ],\n \"text\": \"I like my DRs to be deliverd to me in real time.\",\n \"notifyUrl\":\"http://www.myDomain.com/deliveryReports\"\n }\n ]\n}", "language": "json" } ] } [/block] When your Delivery report is ready, you will receive it on supplied address.
